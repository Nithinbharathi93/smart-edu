## Features

### AI-Powered Course Generation

* **Intelligent Syllabus Creation**: Generate comprehensive courses using PDF materials or simple topic prompts.
* **Adaptive Learning**: Courses are generated specifically for **Beginner, Intermediate, or Advanced** levels.
* **Syllabus Management**: Save, list, and delete courses with automated document context association.

### Integrated Coding Environment

* **Live Code Execution**: Real-time code compilation via the Piston API.
* **Automated Testing**: Submit solutions against **hidden test cases** generated by AI.
* **Multi-Language Support**: Support for Python, JavaScript, and more.

### AI Tutor & Guidance

* **Socratic Hinting**: Get guidance that leads you to the answer without giving it away, based on your configured "Socratic Level."
* **AI Solution Generation**: Generate reference solutions in your preferred programming language.
* **Document-Aware Chat**: Chat directly with your study materials using RAG (Retrieval-Augmented Generation).

### Adaptive Assessment

* **Proficiency Testing**: Take an initial adaptive test to determine your technical level.
* **Weighted Scoring**: Levels are calculated using weighted algorithms to ensure accurate placement.
* **Dynamic Profile**: Your learning experience adapts based on your assessment results.

## üõ† Database Setup & Replication

To ensure local development environments match the production schema exactly, follow these steps to recreate the Supabase tables, RLS policies, and functions.

### 1. Prerequisites

- A [Supabase account](https://supabase.com/) and a fresh project.
- [Supabase CLI](https://supabase.com/docs/guides/cli) installed:

```bash
npm install supabase --save-dev

```

### 2. Environment Linking

Link your local environment to your personal Supabase project:

```bash
npx supabase link --project-ref <your-personal-project-id>

```

_Note: You will be prompted for your Database Password._

### 3. Recreating the Schema

We use a `schema.sql` file (generated via `db dump`) to maintain consistency. Run the following command to execute the script against your project:

```bash
npx supabase db execute --file schema.sql

```

**Alternative (UI Method):**

1. Open your **Supabase Dashboard** -> **SQL Editor**.
2. Copy the entire content of `schema.sql` from this repository.
3. Paste it into a new query and click **Run**.

### 4. Verification

Once the script completes, verify your setup:

- Check the **Table Editor** to ensure all tables (Users, Syllabus, Problems, etc.) are present.
- Ensure **Database Extensions** (like `pgvector` for PDF embeddings) are enabled if the script requires them.

---

### Pro-Tip: Database Migrations

If you make changes to the database structure, please do not edit `schema.sql` manually. Instead, use the migration workflow:

1. Create a migration: `npx supabase db diff -f name_of_change`
2. Push changes: `npx supabase db push`

## API Endpoints

### Authentication

| Method | Endpoint | Description |
| --- | --- | --- |
| **POST** | `/register` | Register user via Supabase |
| **POST** | `/login` | Login and receive JWT access token |

### Document Management

| Method | Endpoint | Description |
| --- | --- | --- |
| **POST** | `/ingest` | Upload PDF, extract text, and store embeddings |
| **GET** | `/list-pdfs` | List all PDFs owned by the user |
| **DELETE** | `/delete-pdf/:id` | Delete a PDF and its associated chunks |

### Syllabus & Courses

| Method | Endpoint | Description |
| --- | --- | --- |
| **POST** | `/generate-syllabus` | Create course from uploaded PDF context |
| **POST** | `/generate-topic-syllabus` | Create course from a text topic |
| **GET** | `/my-courses` | List all generated courses |
| **GET** | `/get-syllabus/:id` | Get full details of a specific syllabus |
| **DELETE** | `/syllabus/:id` | Delete a syllabus and its problems |

### Coding & Execution

| Method | Endpoint | Description |
| --- | --- | --- |
| **POST** | `/generate-problem` | AI generates a coding problem with a repair loop |
| **POST** | `/compile` | Execute code via Piston API |
| **POST** | `/compile/submit` | Run code against hidden AI test cases |
| **POST** | `/problem/solution` | Generate/retrieve AI reference solution |
| **POST** | `/problem/guide` | Get Socratic hints for a specific problem |

### Assessment & Profile

| Method | Endpoint | Description |
| --- | --- | --- |
| **GET** | `/assessment/questions` | Get adaptive test questions |
| **POST** | `/assessment/submit` | Submit test and update user proficiency level |
| **GET** | `/profile/me` | Fetch user profile details |
| **POST** | `/profile/settings` | Update language and Socratic preferences |
